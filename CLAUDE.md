# PU_Identification プロジェクト（再構想版 v2）

## 研究概要

SU（UAV）自身が受信信号から PU（ETC）を認知し、PUが使用中の周波数を特定して、
そこを避けて干渉を与えない周波数を選ぶ —— 認知無線によるUAV緊急ネットワークの動的スペクトルアクセス。
PLATEAUの3D都市環境でレイトレーシングを行い、現実的な伝搬条件下で識別性能を検証する。
最終的に既存のLBT-MAB（Thompson Sampling）に統合する。

---

## 識別の主体と目的（明確化）

```
SUが受信 → 拾った信号が PU(ETC) か 他のSU(UAV) かを判別
        → PUが居る周波数を特定 → そこを避けて干渉しない周波数を選択
```

**なぜPUとSUを「区別」する必要があるのか（研究の動機）:**
RSSIだけでは「電波がある」ことしか分からず、それがPUか他SUか区別できなかった（前回の限界）。
他のSUの信号を誤ってPUと判定すると、本当はPUが居ないのに周波数を回避し、貴重な帯域を無駄にする。
ゆえに「PUか否か」の判別が必須。

## システム設計

### PU（一次ユーザー）：ETC路側機
- 規格：ARIB STD-T75（DSRC）  ※IEEE 802.11p は米国系・別物。混同しないこと
- 周波数：5.8GHz帯　下り 5775–5805MHz（7ch）／上り 5815–5845MHz（7ch）、占有帯域幅 4.4MHz
- 変調：ASK（振幅偏移変調）※DSRC-VICS等の他用途はπ/4 QPSKだが、ETC本体はASK
- 同期パターン：規格で Unique Word が定義 → プリアンブル相関の前提が現実的に成立
- アンテナ：patchMicrostrip（物理モデル）、車道方向への固定指向性
- 送信：車両通過時のバースト通信

### SU（二次ユーザー）：UAV
- 変調：OFDM（サブキャリアは QPSK / 16QAM）。Wi-Fi系（5GHz帯 802.11a/n/ac）を想定
  - 注：OFDMは多重化方式、QPSK/QAMはサブキャリア変調。階層関係（並列ではない）
- アンテナ：dipoleFolded（半波長ダイポール、物理モデル）※完全無指向性ではなくドーナツ型放射
- チャネル選択：LBT + Thompson Sampling（後フェーズで統合）

## SU/PU識別手法

### 主軸（アンテナ1本で計算可能・反射に強い）
| 特徴量 | 内容 | 備考 |
|---|---|---|
| 尖度（Kurtosis） | ASK=振幅が離散的、OFDM=ガウス的 で差が出る | SNRが下がると劣化（後述） |
| プリアンブル相関 | ETCの Unique Word との照合 | マルチパスでも相関ピークは残りやすい |

### AoAは「別建ての比較研究」に切り出す
AoA（到来角）推定には複数素子アレイが必要で、ダイポール1本では不可能。
→ 識別の主軸からは外す。代わりに以下を独立した比較研究テーマとする：

比較シミュレーション計画A：アレイ素子数 vs 識別精度 vs 物理コスト
- 素子数 1 → 2 → 4 → 8 と増やしてAoA識別精度をMATLABでシミュレーション
- 5.8GHzの波長 ≈ 5.2cm、λ/2素子間隔 ≈ 2.6cm → 4素子でも全長8cm弱でドローン搭載可能
- ボトルネックはアンテナ素子でなく受信機チェーン（LNA/ミキサ/ADC）の系統数
  → 重量増・消費電力増（特にADC）・コスト増
- 精度はシミュレーション、重量・消費電力はカタログ値/文献値で見積もり → 総合判断
- アウトプット：「消費電力に見合う素子数の落としどころ」を定量的に示す

## 重要な知見：識別の難所はマルチパスではなくSNR（遮蔽）

簡易シミュレーション（都市相当8タップのマルチパス＋AWGN、尖度のみで識別）の結果：

| SNR | 識別精度 |
|---|---|
| ≥ 0dB | 約90〜97%（反射があっても高精度） |
| -5dB | 約64% |
| -10dB | 約52%（ほぼコイン投げ） |

- 反射（マルチパス）自体は識別をあまり壊さない — ASKの非ゼロ平均成分が尖度に痕跡を残すため
- 本当の脅威は遮蔽によるSNR低下 — 建物の陰で電波が弱まると識別が崩壊する
- → 研究の見せ場は「低SNR領域でいかに精度を保つか」。ここで尖度＋プリアンブル相関の合わせ技が効く
- ※この数値は簡易モデルの目安。本番はMATLABレイトレで実測する

評価の主軸を SNR（遮蔽の度合い）に据える。

## 実装フェーズ

### フェーズ1：信号モデルの構築
ETC信号（ASK）とUAV信号（OFDM）を生成。Toolbox不要、基本MATLABで動く。
※理想信号で識別できるのは当然。これは「コードが動く」確認であって成果ではない。

### フェーズ2：特徴量抽出
尖度・プリアンブル相関を計算するコードを実装。

### フェーズ3：識別器
特徴量を入力にした判定ロジック。まず閾値判定から。

比較シミュレーション計画B：手法 vs SNR
尖度のみ／プリアンブル相関のみ／両者の組み合わせ、を SNR を振って識別精度比較。
低SNRでの優位性を示すのが狙い。評価指標：識別精度、F1、ROC、「90%を超えるSNR閾値」。

### フェーズ4：伝搬環境を段階的に導入（重要：いきなりフル都市に飛ばない）
```
① 自由空間（直接波のみ、反射ゼロ）
② 単純反射を1つ追加（地面2波モデル or 建物1個）
③ フル都市（PLATEAU）
```
段階的にやることで「どの伝搬現象が識別を壊すか」を切り分け（アブレーション実験）。
※PLATEAUデータは plateau.mlit.go.jp から取得（CityGML）

### フェーズ5：LBT-MABとの統合
識別結果をチャネル選択に組み込む。既存研究（PBA-TS by Amrallah et al., 2021）との差分を明確化。
